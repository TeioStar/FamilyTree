# 家谱东方档案感可视化原型设计

## 背景

当前项目已有家谱管理系统的 OpenSpec 方向：一个家谱对应一个 SQLite 文件、家谱级 RBAC、按层加载树图数据。用户本阶段明确选择“视觉原型优先”，目标是在正式系统实现前，先验证一个震撼、可交互、具有东方族谱档案感的家谱可视化工作台。

本原型不接入后端，不实现登录、权限、SQLite 读写或真实文件管理。它通过内置 mock 数据展示未来系统的核心体验，为后续前端工程化和 API 对接提供视觉与交互基准。

## 目标

- 构建一个可直接打开的高质量前端视觉原型。
- 第一屏就是可操作的家谱可视化工作台，而不是介绍页。
- 形成“沉稳、典雅、有纸本族谱和印章质感”的东方族谱档案风格。
- 支持桌面与移动端自适应浏览。
- 支持点击人物、搜索定位、缩放平移、时间轴高亮等核心交互。
- 让数据结构贴近未来后端接口，方便后续迁移到正式 `frontend`。

## 非目标

- 不实现正式后端 API。
- 不实现数据库、登录、RBAC、文件上传。
- 不引入大型构建工具或复杂依赖。
- 不把原型拆成完整工程目录。
- 不追求真实族谱算法的全部边界情况。

## 推荐方案

创建 `prototype/familytree-visual-wow.html`，作为纯前端单页原型。页面内置 CSS、HTML、JavaScript 与 mock 数据，直接在浏览器打开即可运行。

选择单文件原型的原因：

- 视觉探索阶段反馈最快。
- 不要求用户安装依赖或启动服务。
- 可独立保留为视觉基准，不影响后续工程实现。
- 当前仓库尚未形成正式前端结构，避免过早引入脚手架。

## 视觉语言

整体视觉采用“东方族谱档案感”：

- 背景：深色木质案台与宣纸卷轴结合，叠加细微纸纹、淡墨网格和年代刻度。
- 节点：人物节点做成族谱签牌，使用米纸底、墨色文字、细金边和朱印标记。
- 连线：血缘主线使用稳重墨线，配偶关系使用更细的朱砂线。
- 强调：选中人物使用金色描边、淡光晕和印章式状态标记。
- 面板：右侧档案抽屉保持宣纸卡面、竖向信息层级和克制的朱砂强调色。
- 时间轴：底部做成卷轴刻度，用年份和事件节点体现家族时间感。

避免使用：

- 纯科技蓝紫渐变。
- 装饰性漂浮圆球。
- 营销式大标题 hero。
- 过度花哨动画。

## 页面结构

页面分为五个区域：

1. 顶部工具栏
   - 系统名称。
   - 搜索框。
   - 缩放按钮。
   - 回到祖先按钮。
   - 展开层级按钮。
   - 视图切换按钮。

2. 中央家谱画布
   - SVG 绘制人物节点与关系线。
   - 支持平移、缩放和节点点击。
   - 显示主干、配偶、子女和房支标签。

3. 右侧人物档案抽屉
   - 姓名、字辈、性别、出生卒年、籍贯。
   - 父母、配偶、子女关系摘要。
   - 关键事件列表。
   - 资料完整度状态。

4. 底部时间轴
   - 可拖动年份滑块。
   - 高亮该年份相关人物。
   - 显示重要家族事件。

5. 移动端底部抽屉
   - 小屏幕下将右侧档案面板变为底部抽屉。
   - 工具栏收紧，画布保留主要操作能力。

## 交互行为

- 点击人物节点：选中人物，右侧档案面板更新，并高亮父母、配偶、子女。
- 搜索姓名：筛选匹配结果，按回车或点击结果定位到人物。
- 缩放和平移：支持按钮缩放、鼠标滚轮缩放、拖拽画布平移。
- 回到祖先：重置视图到家族始祖和主干分支。
- 展开层级：在 mock 数据中切换显示两代、三代、全部。
- 时间轴滑动：根据年份高亮当时已出生且未去世的人物，并标记相关事件。
- 移动端点击节点：底部抽屉上浮显示人物详情。

## 数据模型

原型内置数据保持接近未来接口：

```js
const persons = [
  {
    id: "p1",
    name: "沈怀远",
    generation: "怀",
    gender: "male",
    birthYear: 1848,
    deathYear: 1912,
    birthplace: "苏州府吴县",
    branch: "长房",
    summary: "族谱主干人物，迁居后重修家乘。",
    completeness: 86
  }
];
```

关系数据：

```js
const relationships = [
  { type: "parent", from: "p1", to: "p3" },
  { type: "spouse", from: "p1", to: "p2" }
];
```

事件数据：

```js
const events = [
  { id: "e1", personId: "p1", year: 1882, type: "migration", title: "迁居吴县" }
];
```

## 布局策略

原型使用固定 mock 数据的分层布局：

- 按 `generation` 或预设 `generationIndex` 计算横向代际位置。
- 同代人物按房支和父级关系排序，保持稳定。
- 配偶节点贴近主节点。
- 子女从父母组向下一代展开。
- SVG 视口通过 `viewBox` 与 CSS 自适应容器大小。

这不是最终生产级树布局算法，但足以验证视觉、信息层级和交互。

## 可访问性与响应式

- 所有按钮使用明确中文标签或可读 `title`。
- 主要交互支持键盘触发搜索。
- 文字不使用随视口缩放的字体大小。
- 移动端避免详情面板遮挡核心画布。
- 节点和按钮保持稳定尺寸，避免 hover 或内容变化造成布局跳动。

## 风险与取舍

- 单文件原型便于快速验证，但不适合长期维护。后续正式实现应迁移到 `frontend` 工程。
- mock 布局不能覆盖超大族谱性能问题。正式版本需要按层加载和虚拟化策略。
- SVG 可满足原型体验，超大规模数据可能需要 Canvas 或 WebGL。
- 原型不接后端，因此权限、审计和真实数据一致性暂不验证。

## 验证策略

- 手动打开 `prototype/familytree-visual-wow.html` 检查首屏视觉。
- 使用浏览器验证桌面宽屏、平板和手机宽度。
- 检查搜索、点击节点、缩放、回到祖先、时间轴高亮是否可用。
- 检查文字是否溢出，主要 UI 是否重叠。
- 确认页面无控制台错误。

## 后续衔接

视觉原型确认后，下一阶段可将其迁移为正式前端：

- `frontend`：React + TypeScript + Vite。
- `backend`：FastAPI + SQLite。
- 可视化数据接口：`nodes`、`edges`、`events`、`has_more`。
- 布局算法从 mock 预设升级为按层加载的可扩展布局。
