## 新增需求

### 需求:家谱数据按文件隔离
系统必须确保每次读写仅作用于请求指定 `family_id` 对应的 SQLite 文件，禁止跨家谱数据访问。

#### 场景:按家谱路由写入
- **当** 客户端提交包含 `family_id` 的人物创建请求
- **那么** 系统必须仅在该 `family_id` 对应数据库写入人物数据

### 需求:核心实体管理
系统必须支持人物、关系、事件、附件四类核心实体的创建、更新与查询。

#### 场景:新增人物并附带事件
- **当** 管理端提交人物资料及出生事件
- **那么** 系统必须成功保存人物记录与事件记录并建立关联

### 需求:关系完整性校验
系统必须禁止明显非法关系，包括人物自关联与重复主关系。

#### 场景:创建自关联关系
- **当** 请求将同一人物同时作为关系两端
- **那么** 系统必须拒绝该请求并返回校验错误

### 需求:删除保护与可追溯
系统必须对删除类操作执行权限校验，并记录删除操作审计信息。

#### 场景:删除人物
- **当** 有权限用户发起人物删除请求
- **那么** 系统必须在执行删除后写入审计日志，包含执行人和目标对象
