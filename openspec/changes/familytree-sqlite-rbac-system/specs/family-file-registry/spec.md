## 新增需求

### 需求:家谱文件扫描与注册
系统必须在启动阶段扫描配置的家谱目录，并识别所有符合命名规则的家谱 SQLite 文件。

#### 场景:启动扫描可用家谱
- **当** 应用完成启动初始化并执行家谱目录扫描
- **那么** 系统必须将每个通过校验的家谱文件注册到可用家谱列表

### 需求:家谱元数据校验
系统必须从每个家谱库读取 `meta` 信息，并禁止注册缺失关键元数据的文件。

#### 场景:元数据缺失
- **当** 某家谱文件缺失 `family_id` 或 `schema_version`
- **那么** 系统必须将该文件标记为不可用并记录原因

### 需求:单库故障隔离
系统必须保证单个家谱文件损坏或迁移失败时，不影响其他家谱库可用。

#### 场景:存在损坏家谱文件
- **当** 扫描过程中发现某家谱文件无法打开或校验失败
- **那么** 系统必须跳过该文件并继续注册其余有效家谱

### 需求:家谱列表查询
系统必须提供可查询的家谱注册结果，至少包含 `family_id`、`name`、`status` 与错误摘要。

#### 场景:查询家谱列表
- **当** 客户端请求家谱列表
- **那么** 系统必须返回当前注册状态和不可用原因（如有）
