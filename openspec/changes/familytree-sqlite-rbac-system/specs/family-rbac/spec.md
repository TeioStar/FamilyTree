## 新增需求

### 需求:固定角色模型
系统必须实现家谱级固定角色 `creator`、`admin`、`member`，并禁止使用未定义角色进行授权。

#### 场景:创建成员关系时角色非法
- **当** 请求分配不在角色集合内的角色
- **那么** 系统必须拒绝该请求并返回角色非法错误

### 需求:创建者不可被管理员降级
系统必须禁止 `admin` 用户降级、移除或禁用 `creator` 用户。

#### 场景:管理员尝试移除创建者
- **当** `admin` 发起移除 `creator` 的成员关系请求
- **那么** 系统必须拒绝该请求并记录越权审计

### 需求:关键操作强审计
系统必须对角色变更、批量删除和家谱删除进行强制审计记录。

#### 场景:角色变更
- **当** 任一用户角色发生变化
- **那么** 系统必须记录 `who/when/from/to` 等审计字段

### 需求:最小创建者约束
系统必须保证每个家谱始终至少存在 1 名 `creator`。

#### 场景:移除最后一个创建者
- **当** 请求将家谱中的最后一个 `creator` 移除或降级
- **那么** 系统必须拒绝该请求并提示违反最小创建者约束
