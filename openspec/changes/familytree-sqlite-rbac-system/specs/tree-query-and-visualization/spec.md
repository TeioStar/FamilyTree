## 新增需求

### 需求:树图按层加载
系统必须提供以 `root_person_id` 和 `depth` 为参数的树查询能力，禁止默认返回无限深度全量树。

#### 场景:请求指定深度树
- **当** 客户端请求根节点为 A、深度为 3 的家谱树
- **那么** 系统必须仅返回深度 3 以内的节点与边

### 需求:可续取结果标记
系统必须在树结果被截断时返回明确的续取信号（如 `has_more` 或续取游标）。

#### 场景:存在未展开分支
- **当** 查询深度不足以覆盖完整子树
- **那么** 系统必须在响应中标记仍可继续加载的分支

### 需求:关系图基础契约
系统必须返回可直接用于前端可视化的节点与边结构，至少包含节点唯一标识、显示名称和关系类型。

#### 场景:获取关系图数据
- **当** 客户端请求某家谱的关系图视图数据
- **那么** 系统必须返回结构化的 `nodes` 与 `edges` 集合

### 需求:查询稳定排序
系统必须对同级节点使用稳定排序规则，保证同一输入条件下返回顺序一致。

#### 场景:重复请求同一子树
- **当** 客户端在未发生数据变化时重复请求同一树片段
- **那么** 系统必须返回一致的同级节点顺序
