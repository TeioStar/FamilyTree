# FamilyTree UI 设计方案（MVP）

## 1. 设计目标

- 面向 Web 管理端，服务对象为 `creator`、`admin`、`member` 三类用户。
- 与当前 OpenSpec 对齐，优先覆盖四个核心能力：
  - 家谱文件扫描与注册状态查看
  - 树图按层加载与续取
  - 家谱级 RBAC 管理
  - 关键操作审计可追溯
- 优先保证信息清晰、可操作、可回溯，不追求复杂视觉特效。

## 2. 视觉方向

- 风格：`Data-Dense Dashboard` + `Accessible & Ethical`
- 关键词：可信、克制、清晰、层次分明
- 品牌基调：专业蓝为主、琥珀色强调风险提示、绿色表示成功状态

## 3. 信息架构

1. 家谱总览（默认页）
2. 树图浏览
3. 成员与权限
4. 审计日志
5. 系统健康（可选，后续）

全局元素：

- 顶部栏：当前家谱切换、搜索、当前用户角色、高风险操作入口
- 左侧导航：页面分区
- 主内容区：卡片化模块

## 4. 核心页面设计

## 4.1 家谱总览

### 目标

- 展示扫描后的家谱列表与可用状态。
- 支持用户快速切换家谱，明确不可用原因。

### 主要模块

- 统计卡片：总家谱数 / 可用数 / 不可用数
- 列表表格（关键列）：
  - `family_id`
  - 家谱名称
  - 状态（可用 / 不可用）
  - 错误摘要（仅不可用时展示）
  - 最后扫描时间
- 操作：
  - 设为当前家谱
  - 重新扫描（全局）

### 关键交互

- 若家谱不可用，行内展示错误标签，禁用“进入管理台”动作。
- 当前家谱切换后，右上角显示上下文标签：`当前家谱：xxx`。

## 4.2 树图浏览

### 目标

- 支持 `root_person_id + depth` 的按层加载，不一次性渲染整树。
- 明确“还有更多节点”的续取信号。

### 主要模块

- 过滤区：
  - 根节点选择
  - 深度选择（1-5）
  - 查询按钮
- 可视化区：
  - 节点卡片（姓名、生卒、关系标签）
  - 连接线（首版可简化）
  - 分支 `has_more` 标识
- 续取区：
  - “加载下一层”按钮
  - 续取状态提示（已加载到第 N 层）

### 关键交互

- 请求返回 `has_more=true` 时，展示“加载下一层”按钮。
- 同级节点按稳定规则展示，重复查询顺序不变。

## 4.3 成员与权限（RBAC）

### 目标

- 明确角色矩阵，降低误操作。
- 在 UI 层提前阻断明显违规动作。

### 主要模块

- 角色说明条：
  - `creator`: 家谱所有权与最终管理权
  - `admin`: 成员管理与日常维护
  - `member`: 内容协作
- 成员表格：
  - 用户名
  - 当前角色
  - 修改角色
  - 移除成员
  - 约束提示

### 关键交互规则（与 spec 对齐）

- `admin` 不可降级/移除 `creator`（控件禁用 + 悬浮提示）。
- 最后一个 `creator` 不可降级或移除（控件禁用 + 明确提示）。
- 角色变更成功后，显示 toast，并写入审计流。

## 4.4 审计日志

### 目标

- 对高风险操作形成可检索、可筛选的证据链。

### 主要模块

- 过滤条件：操作类型、操作者、时间范围
- 日志时间线：
  - `when`
  - `who`
  - `action`
  - `target`
  - 变更字段（如 `from -> to`）

### 高风险动作二次确认

- 适用操作：批量删除、家谱删除、角色重大变更。
- 确认方式：弹窗 + 输入确认词（例如 `DELETE`）。

## 5. 状态设计（必须覆盖）

- `Loading`：骨架屏 + 禁用提交按钮。
- `Empty`：友好引导 + 主操作按钮（如“创建首位人物”）。
- `Error`：错误摘要 + 重试入口 + 不暴露内部堆栈。
- `Success`：操作完成提示（3 秒自动消失）。

## 6. 响应式与可访问性

- 断点建议：
  - Desktop: `>= 1200px`（左侧栏常驻）
  - Tablet: `768-1199px`（左侧栏折叠）
  - Mobile: `< 768px`（顶部 Tab 导航）
- 无障碍要求：
  - 文本对比度不低于 4.5:1
  - 可见焦点环（至少 2px）
  - 所有关键操作可键盘触达
  - 表单控件具备可读标签
  - 状态提示使用 `aria-live`

## 7. 页面文案（首版）

- 家谱不可用：`该家谱文件不可用，请查看错误摘要并联系维护者。`
- 最后创建者保护：`至少保留 1 名创建者，当前操作已被阻止。`
- 越权提示：`当前角色无权执行此操作。`
- 续取提示：`当前结果已截断，可继续加载下一层。`

## 8. 验收标准（UI 维度）

1. 用户可以在 3 次点击内完成家谱切换并进入对应视图。
2. RBAC 违规操作在前端可见且不可提交。
3. 树图支持按层加载，并能直观看到 `has_more`。
4. 审计日志可看到角色变更 `who/when/from/to`。
5. 移动端在 390px 宽度下可完整完成浏览和基础管理。
